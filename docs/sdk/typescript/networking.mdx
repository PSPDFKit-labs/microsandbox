## NetworkPolicy

Static factory for creating preset network policies.

---

#### NetworkPolicy.allowAll()

```typescript
static allowAll(): NetworkConfig
```

Unrestricted network access, including to private addresses and the host machine.

**Returns**

| Type | Description |
|------|-------------|
| [`NetworkConfig`](#networkconfig) | Network configuration |

---

#### NetworkPolicy.none()

```typescript
static none(): NetworkConfig
```

Deny all traffic. No network interface is created - the guest is fully offline. `exec` and `fs` still work since they use the host-guest channel, not the network.

**Returns**

| Type | Description |
|------|-------------|
| [`NetworkConfig`](#networkconfig) | Network configuration |

---

#### NetworkPolicy.publicOnly()

```typescript
static publicOnly(): NetworkConfig
```

Block private address ranges and cloud metadata endpoints. Allow everything else. This is the **default** policy.

**Returns**

| Type | Description |
|------|-------------|
| [`NetworkConfig`](#networkconfig) | Network configuration |

## Types

### NetworkConfig

Network configuration object passed as [`SandboxConfig.network`](/sdk/typescript/sandbox#sandboxconfig). Use [`NetworkPolicy`](#networkpolicy) presets for common cases, or build a custom configuration with rules.

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| defaultAction? | [`PolicyAction`](#policyaction) | `'allow'` | Action when no rule matches |
| dns? | [`DnsConfig`](#dnsconfig) | - | DNS interception configuration |
| maxConnections? | `number` | - | Maximum concurrent connections |
| policy? | `string` | - | Preset name (set automatically by `NetworkPolicy.*()`) |
| rules? | `Array<`[`PolicyRule`](#policyrule)`>` | `[]` | Custom rules evaluated first-match-wins |
| tls? | [`TlsConfig`](#tlsconfig) | - | TLS interception configuration |
| trustHostCas? | `boolean` | `false` | Ship the host's trusted root CAs into the guest at boot so outbound TLS works behind corporate MITM proxies (Warp Zero Trust, Zscaler, etc.). Opt-in |
| egressInterceptHosts? | `Array<string>` | `[]` | Hosts to intercept. `"*"` for all. Supports exact match and `*.suffix` wildcards. Empty = no interception. See [Egress Interception](/networking/egress) |
| egressMaxBodyBytes? | `number` | `67108864` | Max body bytes captured per request/response (default: 64 MiB) |
| egressInterceptTimeoutMs? | `number` | `5000` | SDK decision timeout in milliseconds. Fail-open on timeout |
| egressTimeoutMs? | `number` | `300000` | Per-connection wall-clock timeout in milliseconds. Prevents SSE/streaming hangs. `0` to disable |

### DnsConfig

DNS interception settings.

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| blockDomains? | `Array<string>` | `[]` | Block DNS for exact domains (returns REFUSED) |
| blockDomainSuffixes? | `Array<string>` | `[]` | Block DNS for all subdomains of a suffix |
| rebindProtection? | `boolean` | `true` | Block DNS responses resolving to private IPs |
| nameservers? | `Array<string>` | `[]` | Nameservers (`IP`, `IP:PORT`, `HOST`, or `HOST:PORT`). Overrides the host's `/etc/resolv.conf` when set. Hostnames are resolved once at startup via the host's OS resolver |
| queryTimeoutMs? | `number` | `5000` | Per-DNS-query timeout in milliseconds |

### PolicyAction

| Value | Description |
|-------|-------------|
| `'allow'` | Permit the traffic |
| `'deny'` | Drop the traffic silently |

### PolicyDirection

| Value | Description |
|-------|-------------|
| `'inbound'` | Traffic entering the sandbox (via published ports) |
| `'outbound'` | Traffic leaving the sandbox |

### PolicyProtocol

| Value | Description |
|-------|-------------|
| `'icmpv4'` | ICMPv4 traffic |
| `'icmpv6'` | ICMPv6 traffic |
| `'tcp'` | TCP traffic |
| `'udp'` | UDP traffic |

### PolicyRule

A single network policy rule.

| Field | Type | Description |
|-------|------|-------------|
| action | [`PolicyAction`](#policyaction) | What to do when this rule matches |
| destination? | `string` | Target filter: `'any'`, `'private'`, `'loopback'`, `'metadata'`, `'multicast'`, `'link-local'`, or a CIDR like `'10.0.0.0/8'` |
| direction? | [`PolicyDirection`](#policydirection) | Traffic direction |
| port? | `string` | Single port (`"443"`) or range (`"8000-9000"`) |
| protocol? | [`PolicyProtocol`](#policyprotocol) | Protocol filter |

### TlsConfig

TLS interception settings within [`NetworkConfig`](#networkconfig).

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| blockQuic? | `boolean` | `true` | Block QUIC/HTTP3 (UDP) on intercepted ports, forcing TCP/TLS fallback |
| bypass? | `Array<string>` | `[]` | Domains to skip interception. Supports `*.suffix` wildcards. Use for domains with certificate pinning. |
| interceptCaCert? | `string` | - | Path to a custom interception CA certificate PEM file |
| interceptCaKey? | `string` | - | Path to a custom interception CA private key PEM file |
| interceptedPorts? | `Array<number>` | `[443]` | TCP ports where TLS interception is active |
| upstreamCaCert? | `Array<string>` | `[]` | Paths to additional trusted CA PEM files for upstream verification |
| verifyUpstream? | `boolean` | `true` | Verify upstream server certificates. Set to `false` only for self-signed servers. |

## Egress Interception

See [Egress Interception](/networking/egress) for conceptual overview.

---

#### egressIntercept()

```typescript
async function egressIntercept(
    sandbox: Sandbox,
    options: EgressInterceptOptions
): Promise<void>
```

Run egress interception with callback hooks. Blocks until the sandbox stops or the connection closes.

**Parameters**

| Parameter | Type | Description |
|-----------|------|-------------|
| sandbox | `Sandbox` | A running sandbox with `egressInterceptHosts` configured |
| options | [`EgressInterceptOptions`](#egressinterceptoptions) | `onRequest` and/or `onResponse` hooks |

---

### EgressInterceptOptions

| Field | Type | Description |
|-------|------|-------------|
| onRequest? | `(request, ctx) => EgressHttpRequest \| EgressHttpResponse \| undefined` | Called for each outbound request. Return modified request, response (short-circuit), `undefined` (pass through), or throw to block |
| onResponse? | `(response, request, ctx) => EgressHttpResponse \| undefined` | Called for each server response. Return modified response, `undefined` (pass through), or throw to block |

### EgressHttpRequest

| Field | Type | Description |
|-------|------|-------------|
| method | `string` | HTTP method |
| uri | `string` | Request URI |
| headers | `Array<[string, string]>` | Header name-value pairs |
| body? | `Buffer` | Request body (if captured) |

### EgressHttpResponse

| Field | Type | Description |
|-------|------|-------------|
| status | `number` | HTTP status code |
| headers | `Array<[string, string]>` | Header name-value pairs |
| body? | `Buffer` | Response body (if captured) |

### EgressContext

Connection metadata passed to hook callbacks.

| Field | Type | Description |
|-------|------|-------------|
| sni | `string` | TLS Server Name Indication |
| dst | `string` | Destination address |
| connectionId | `number` | Connection ID |
| timestampMs | `number` | Event timestamp |
